## Introduction

Schedules can be used to run any selection of your tables at a user-specified frequency. Running your scripts on a repeating schedule ensures output data is always kept up to date.

<div className="pt-callout pt-icon-info-sign pt-intent-warning" markdown="1">
  Schedules <b>only run from the `production` workspace</b>. Make sure you commit and push your
  changes to the production workspace before configuring schedules.
</div>

## Use schedules

1. Navigate to the Scheduling page using the left menu
2. Click `Add schedule`
3. Enter your schedule's settings and click `Create schedule`

  src="/static/images/platform_guides/scheduling/create_schedule.png"
  style={{ width: "100%", padding: "20px" }}
/>

## View past runs

You can find all current and historical scheduled runs in the _Run logs_ page, accessible using the left menu bar.
