# Extreme weather JiT showcase

This example demonstrates the power of Dataform's **Just-in-Time (JiT) compilation**.

Unlike standard Ahead-of-Time (AoT) compilation, where all SQL is generated statically during the `compile` phase, JiT allows actions to contain dynamic JavaScript code (`jitCode`) that can be re-evaluated just before execution.

## Key capabilities demonstrated

### 1. Dynamic metric explorer (`jit_weather_analysis.js`)
The JiT logic programmatically selects columns and filters based on a list of weather conditions passed via `vars`.
- **Logic**: Iterates over a `metrics` array (e.g., `snow,thunder,tornado`) to generate `SUM(CAST(metric AS INT64))` columns and an `OR` filter in the `WHERE` clause.
- **Value**: Shows how JiT can generate the **shape of the SQL** based on input data.

### 2. Unit-aware threshold analysis (`jit_weather_analysis.js`)
Demonstrates conditional math and formula injection.
- **Logic**: Accepts a `tempUnit` (C/F) and `tempThreshold`. It injects either raw values or the mathematical conversion formula `((max_temp - 32) * 5 / 9)` directly into the `SELECT` and `WHERE` clauses.
- **Value**: Shows how to **abstract away complex SQL formulas**, keeping the warehouse-side code optimized.

### 3. Dynamic grouping (`jit_regional_summary.js`)
Changes the core aggregation level of the query.
- **Logic**: Branches the `SELECT` and `GROUP BY` logic based on the `groupByMonth` variable.
- **Value**: Demonstrates **structural branching** that goes beyond simple variable replacement.

## Demonstration

You can see the JiT compilation in action by passing different variables to the compile command:

### Metric selection and unit conversion
```bash
dataform compile --vars=metrics=snow,hail,tempUnit=C,tempThreshold=25 --json
```
The output `jitCode` will show SQL columns for `snow` and `hail`, and Celsius conversion formulas.

### Granularity switching
```bash
dataform compile --vars=groupByMonth=true --json
```
The `jit_regional_summary` action will show `DATE_TRUNC` logic in its `jitCode`.

## Benefits of JiT

1. **Dynamic logic**: Queries adapt to external parameters without source code changes.
2. **Structural flexibility**: Generate columns, joins, or grouping clauses programmatically.
3. **Optimized execution**: Offload complex conditional logic from the warehouse by generating specific, ready-to-run SQL.

## Experiencing the full JiT lifecycle

While the `compile` command allows you to inspect the dynamic logic generated by JiT, the full power is experienced during execution.

When you **run** a project containing JiT actions, Dataform re-evaluates the `jitCode` just before sending the SQL to the warehouse. This ensures that the most up-to-date variables and warehouse state can influence the execution.

Try running the analysis with your own custom thresholds:
```bash
dataform run --vars=metrics=snow,tempUnit=C,tempThreshold=30
```
This will execute the dynamically generated SQL directly in your BigQuery project.
